```

   The bible of distributed systems

2. "System Design Interview" (Vol 1 & 2) - Alex Xu
   Practical system design patterns

   Microservices architecture guide

   Software architecture principles

### Software Engineering

```
1. "Clean Code" - Robert Martin
   Writing readable code

2. "The Pragmatic Programmer" - Hunt & Thomas
   Practical programming wisdom

3. "Refactoring" - Martin Fowler
   Improving existing code

4. "Code Complete" - Steve McConnell
   Comprehensive construction guide

## üí° Pro Tips

### Learning Effectively

```
1. Build, don't just read
   - Implement what you learn
   - Break things, fix them

2. Teach others
   - Write blog posts
   - Give presentations
   - Mentor juniors

3. Read source code
   - Open source projects
   - Your company's codebase

4. Join communities
   - Discord servers
   - Local meetups
   - Conferences
```

### Career Growth

```
1. Seek feedback actively
   - Regular 1:1s with manager
   - Peer feedback

2. Take on challenges
   - Volunteer for hard problems
   - Learn from failures

3. Build relationships
   - Cross-team collaboration
   - Industry network

4. Document your wins
   - Keep a brag document
   - Track impact metrics
```
